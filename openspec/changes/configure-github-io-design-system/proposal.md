## Why

The showcase needs a consistent, accessible visual foundation before feature UI is built. Adding Tailwind v4 and shadcn/ui early lets later changes compose production-facing components without repeatedly revisiting styling infrastructure.

## What Changes

- Configure Tailwind v4 for the generated Vite React app.
- Initialize shadcn/ui with the `new-york` style and app-local component source.
- Add a quiet professional theme foundation with light, dark, and system modes.
- Add a visible theme toggle with persisted preference.
- Keep skill data, search, final showcase composition, and deployment out of scope.

## Capabilities

### New Capabilities

- `github-io-design-system`: Covers Tailwind v4, shadcn/ui setup, theme tokens, and light/dark/system theme behavior.

### Modified Capabilities

None.

## Impact

- Adds Tailwind and shadcn dependencies/configuration to `apps/github.io`.
- Adds app-owned UI component source generated by shadcn.
- Adds theme utilities and tests used by later UI changes.
