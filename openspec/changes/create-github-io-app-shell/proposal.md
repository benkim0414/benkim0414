## Why

The repository needs a dedicated React application shell before the personal GitHub Pages skills showcase can be implemented. Creating the app through Nx establishes the build, test, and project boundaries that later OpenSpec changes can extend safely.

## What Changes

- Generate a new Nx React app named `github.io` at `apps/github.io`.
- Use Vite, TypeScript, Vitest, and Playwright as the generated app stack.
- Configure the app for static output suitable for the root `benkim0414.github.io` site.
- Keep the app shell minimal; no showcase content, shadcn design system, search, or deployment automation is included in this change.

## Capabilities

### New Capabilities

- `github-io-app-shell`: Covers the generated React app shell, Nx targets, static build output, and initial smoke tests.

### Modified Capabilities

None.

## Impact

- Adds a new Nx project under `apps/github.io` and its generated E2E project.
- Adds React/Vite/testing dependencies required by the Nx generator.
- Updates pnpm and Nx workspace metadata generated by Nx.
- Implementation must happen from an isolated worktree and commit this change atomically before dependent changes.
